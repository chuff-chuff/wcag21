達成基準 2.5.1 の失敗例 － 単純なポインタによる代替なしに、軌跡ベースのジェスチャによる機能を提供している

達成方法に関する重要な情報

これらの達成方法 (参考) の使用法及び、それらが WCAG 2.1 達成基準 (規定) とどのように関係するかに関する重要な情報については、WCAG 達成基準の達成方法を理解するを参照のこと。適用 (対象) セクションは、その達成方法の範囲について説明しており、特定の技術に関する達成方法があるからといって、WCAG 2.1 を満たすコンテンツを作成する際に、常にその技術が使用可能であるわけではない。

適用 (対象)

ポインタを使用してトリガーすることができるインタラクティブなコントロールを含むすべての技術。

これは達成基準2.5.1: ポインタのジェスチャ (失敗例) に関する達成方法である。

解説

この失敗例の目的は、軌跡ベースのジェスチャで操作される機能をコンテンツ制作者が実装し、かつ、単純なポインタのジェスチャでこの機能を操作する代替手段が存在しない状況を記述することである。

コンテンツ及び機能を作成しなかった場合、軌跡ベースのジェスチャは、タッチスクリーン上でコンテンツを探索したり、又はページのコードをチェックして touchstart や touchend のような特定のイベントハンドラが存在するかどうかを確認することで、見つけることができる。軌跡ベースのジェスチャの詳細については、ポインタのジェスチャの解説書を参照のこと。

注記: 軌跡ベースのジェスチャで実装された機能については、キーボードでも操作可能であることは有益である（達成基準 2.1.1 キーボード を満たすのに役立つかもしれない）。しかし、達成基準 2.5.1のポイントは、多くのデバイスでキーボードを使用できないポインタの利用者が、単純なポインタによる入力で機能を操作できる代替手段を確保できるようにすることにある。

事例

例 1

  • A web application implements an interface where drawing a "Z" on the screen undoes the last action. There is no other way to undo the action.
  • A swipe-to-reveal control displays a set of options when swiping an item to the left, and another set of options when swiping an item to the right. One or more of these options are not available after the item is first opened with a single tap or click.

検証

手順

For each part of the content that implements a function that can be activated by a path-based gesture:

  1. Check that controls are available through which the same function can be carried out via simple taps or clicks.

期待される結果

  • If check手順 #1 が偽である場合、コンテンツは達成基準を満たさないことになる。
